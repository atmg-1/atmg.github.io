## 🚀 快速开始

### 环境准备

确保已安装以下工具：
- **Node.js** (推荐 v16+)
- **Git** 
- **Hexo CLI**

```bash
# 检查 Node.js 版本
node -v

# 检查 Git 版本
git --version

# 检查 Hexo 版本
hexo version
```

### 进入博客目录

```bash
cd d:\boke\bolg
```

### 首次使用

```bash
# 安装依赖
npm install

# 清理缓存
hexo clean

# 生成静态文件
hexo generate

# 启动本地服务器
hexo server

# 访问 http://localhost:4000
```

## ✍️ 发布新文章

### 方法一：交互式脚本（最简单）

```bash
.\publish_blog.bat
```

**脚本会自动提示输入：**
- ✅ 文章标题
- ✅ 文章内容
- ✅ 标签（用逗号分隔）
- ✅ 分类

**然后自动完成：**
1. 创建文章文件
2. 生成静态页面
3. 部署到 GitHub

---

### 方法二：命令行创建（推荐）

#### 步骤 1：创建文章

```bash
hexo new post "文章标题"
```

**示例：**
```bash
hexo new post "我的第一篇技术文章"
```

#### 步骤 2：编辑文章

文章会创建在 `source\_posts\文章标题.md`

用编辑器打开，填写内容：

```markdown
---
title: 我的第一篇技术文章
date: 2026-03-24 12:00:00
tags: [技术，编程，Hexo]
categories: [技术分享]
---

这里是文章正文...

## 第一部分

内容...

## 第二部分

内容...
```

#### 步骤 3：本地预览

```bash
# 清理并生成
hexo clean && hexo generate

# 启动服务器
hexo server

# 访问 http://localhost:4000 查看效果
```

#### 步骤 4：发布到网上

```bash
# 方式一：使用完整发布脚本（推荐）
.\full_publish.bat

# 方式二：手动部署
hexo clean
hexo generate
hexo deploy
```

---

### 方法三：快速创建脚本

创建 `quick_post.bat` 文件：

```batch
@echo off
set /p title="请输入文章标题："
hexo new post "%title%"
echo.
echo 文章已创建：source\_posts\%title%.md
echo 请用编辑器打开并编辑该文件
echo.
pause
```

## 🐛 常见问题解决

### 问题 1：文章发布后没有显示

**解决方案：**

```bash
# 1. 清理缓存
hexo clean

# 2. 重新生成
hexo generate

# 3. 重新部署
hexo deploy

# 4. 等待 5 分钟让 GitHub Pages 更新
```

**检查清单：**
- [ ] 文章 Front-Matter 是否正确
- [ ] 文章是否保存
- [ ] 部署是否成功
- [ ] GitHub Actions 是否运行完成

---

### 问题 2：本地预览正常，部署后异常

**可能原因：**
- 路径配置错误
- CDN 缓存问题
- 文件未正确提交

**解决方案：**

```bash
# 检查 _config.yml 中的 URL
url: https://atmg-1.github.io/atmg.github.io

# 清除浏览器缓存
Ctrl + Shift + Delete

# 使用无痕模式测试

# 检查 Git 提交
git status
git add .
git commit -m "fix: 修复部署问题"
git push origin main
```

---

### 问题 3：样式显示不正常

**解决方案：**

```bash
# 1. 检查 Stylus 文件语法
# 编辑 source/_data/*.styl 文件

# 2. 清理并重新生成
hexo clean && hexo generate

# 3. 强制刷新浏览器
Ctrl + F5

# 4. 检查 CSS 是否生成
查看 public/css/ 目录是否有 CSS 文件
```

---

### 问题 4：GitHub Actions 部署失败

**解决方案：**

1. 访问 https://github.com/atmg-1/atmg.github.io/actions
2. 查看最新的 workflow run
3. 点击查看详情和错误日志
4. 根据错误信息修复

**常见错误：**
- `hexo command not found` → 需要先安装依赖
- `git push failed` → 检查 Git 配置
- `Permission denied` → 检查 token 权限

---

### 问题 5：图片无法显示

**解决方案：**

```bash
# 检查图片路径
# 图片应该放在 source/images/ 目录

# 在文章中使用正确的路径
![描述](/images/photo.jpg)

# 不是相对路径
![描述](../images/photo.jpg) ❌
```

## 🎓 高级定制

### 1. 创建自定义页面

```bash
# 创建关于页面
hexo new page about

# 创建标签页面
hexo new page tags

# 创建分类页面
hexo new page categories

# 创建友情链接页面
hexo new page links
```

生成的文件在 `source/页面名/index.md`，可以编辑内容。

---

### 2. 添加友情链接

在 `source/_data/` 目录创建 `links.yml`：

```yaml
- title: 朋友 A 的博客
  url: https://example.com
  avatar: /images/avatar-a.png
  description: 技术大牛

- title: 朋友 B 的博客
  url: https://example.org
  avatar: /images/avatar-b.png
  description: 设计达人
```

然后在主题配置中启用。

---

### 3. 启用评论系统

编辑 `_config.next.yml`：

```yaml
# Gitalk 评论
gitalk:
  enable: true
  github_id: atmg-1
  repo: atmg.github.io
  client_id: your_client_id
  client_secret: your_client_secret
  admin_user: atmg-1
  distraction_free_mode: false
```

**获取 Client ID 和 Secret：**
1. 访问 https://github.com/settings/developers
2. 创建 OAuth App
3. 填写 Application name 和 Homepage URL
4. 复制 Client ID 和 Client Secret

---

### 4. 添加搜索功能

编辑 `_config.yml`：

```yaml
# 本地搜索
search:
  path: search.json
  field: post
  format: html
  limit: 10000
```

主题会自动启用搜索功能。

---

### 5. 启用数学公式

编辑 `_config.yml`：

```yaml
# MathJax 支持

    enable: true
```

```markdown

$$
```

### 文件位置速查

| 类型 | 位置 | 说明 |
|------|------|------|
| 文章 | `source/_posts/` | Markdown 格式 |
| 页面 | `source/` | 独立页面 |
| 自定义 JS | `source/js/` | 特效脚本 |
| 自定义 CSS | `source/_data/` | 样式文件 |
| 图片资源 | `source/images/` | 图片文件 |
| 配置文件 | `_config.yml` | 站点配置 |
| 主题配置 | `_config.next.yml` | 主题配置 |
| 模板 | `scaffolds/` | 文章模板 |
| 生成的文件 | `public/` | 静态文件（不要手动修改） |

---

### Front-Matter 模板

```yaml
---
title: 文章标题              # 必填
date: 2026-03-24 12:00:00   # 必填（自动生成）
updated: 2026-03-24 12:00:00 # 可选（更新时间）
tags:                       # 可选（多个标签）
  - 技术
  - 编程
categories:                 # 可选（多级分类）
  - 技术分享
cover: /images/cover.jpg    # 可选（封面图片）
toc: true                   # 可选（是否显示目录）
mathjax: true               # 可选（是否启用数学公式）
draft: false                # 可选（是否为草稿）
---
```

## 💡 最佳实践

### ✅ 应该做的

1. 每次写作前先 `hexo clean`
2. 定期备份整个博客目录
3. 部署前在本地测试
4. 保持依赖更新
5. 监控 GitHub Actions 状态
6. 使用 Git 版本控制
7. 文章及时添加标签和分类

### ❌ 不应该做的

1. 不要手动修改 `public/` 目录
2. 不要删除 `.deploy_git/` 目录
3. 不要提交敏感信息（API Key 等）
4. 不要在高峰期频繁部署
5. 不要使用过大的图片
6. 不要忘记本地测试直接部署
